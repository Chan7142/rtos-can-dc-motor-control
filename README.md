# Embedded-System-From-Scratch

From Register-level Control to Communication Protocols(CAN/SPI/I2C) \& RTOS. A deep dive into embedded software engineering.



📅 Roadmap (학습 계획표)

🏁 Phase 1: Preparation \& Basic Communication (준비 및 기초 통신)

4/10 ~ 4/20: Hardware Setup \& Environment (기본 키트 구매 및 환경 설정)

Set up Development environment with Arduino Uno, Nucleo-F401RE.

Hardware: Tilt Sensor, MPU-6050, Logic Analyzer, etc.



4/21 ~ 4/30: SPI Protocol Implementation (SPI 통신 구현)

MCU: Arduino, Nucleo

Sensor: ADXL345 (3-axis Accelerometer)



5/01 ~ 5/10: I2C Protocol Implementation (I2C 통신 구현)

MCU: Arduino, Nucleo

Sensor: MPU-6050 (Gyro/Accelerometer)



🚀 Phase 2: Deep Dive into Hardware (하드웨어 심화)

5/11 ~ 6/15: Register-level Control \& Driver Development (레지스터 제어 및 드라이버 제작)

MCU: Nucleo (STM32)

Details: Developing custom drivers for all sensors at the register level without high-level libraries.



6/16 ~ 7/15: CAN Bus Communication (CAN 통신 구현)

MCU: Arduino, Nucleo

Components: MCP2551 (CAN Transceiver) x 2, 120Ω Resistors.

Goal: Implementing automotive standard communication protocol.



🤖 Phase 3: System Architecture (시스템 설계)

7/16 ~ 8/16: RTOS System Design (RTOS 시스템 설계)

MCU: Nucleo (STM32)

Components: LEDs, Buttons, 330Ω Resistors.

Details: Multi-tasking management using Real-Time Operating System.



8/17 ~ : Final Review \& Wrap-up (마무리 및 복습)
